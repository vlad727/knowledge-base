#### Команда для проверки I/O активности:
iostat -x 1
Эта команда покажет статистику использования дисков. 
Обратите внимание на столбцы %util (процент использования диска) и await (время ожидания ввода-вывода). 
Высокое значение await может указывать на проблемы с дисковой подсистемой.

#### Команда для проверки статуса SMART:
smartctl -a /dev/sda | grep -E 'Reallocated_Sector_Ct|Current_Pending_Sector|Offline_Uncorrectable'
Здесь /dev/sda заменяется на имя вашего диска. 
Эта команда покажет количество переназначенных секторов и ожидающих обработки ошибок. 
Большое число таких секторов указывает на возможные проблемы с диском.

### Шаг 2: Анализ загрузки CPU
Следующий шаг — анализ того, какие процессы потребляют ресурсы CPU.

#### Команда для просмотра процессов:
top -b -n1 | head -n20
Это позволит увидеть список наиболее ресурсоемких процессов. 
Обратите внимание на колонку %CPU.

#### Команда для анализа steal time:
mpstat -P ALL 1
Команда mpstat показывает информацию о времени простоя (idle), пользовательском времени (usr) и времени, украденном гипервизором (st). 
Если значение st велико, это может свидетельствовать о том, что гипервизор забирает ресурсы у вашей виртуальной машины.

#### Команда для мониторинга сетевых интерфейсов:
Bash

sar -n DEV 1
Эта команда покажет сетевую активность по каждому интерфейсу. 
Высокая загрузка сети может оказывать влияние на общую производительность.
