Issue:
Daemonset won't start and failed with error crashLoopBakc

Errors:
No errors, events also does not contain any errors

Root Cause:
# Settings for daemonset below
hostPID: true
hostIPC: true
hostNetwork: true

and node-local-dns already bind such port

Troubleshoting:
After delete such setting I get an errors below
"listen tcp4 :8080: bind: address already in use"

Explanation:
"hostNetwork: true" means that the pod uses the host node's network namespace directly, bypassing the virtual network provided by the cluster
when you use such parameter you don't need service, you can do request by "node-ip:port-daemonset"

with mount and volumes you will see PID's but can't kill it, because kill search pid's by current container process
/app # kill -SIGTERM  1058115
sh: can't kill pid 1058115: No such process
so you need use hostPID true
          volumeMounts:
            - mountPath: "/proc"
              name: proc-dir
            - mountPath: "/sys"
              name: sysfs
      volumes:
        - name: proc-dir
          hostPath:
            path: /proc

with prviliged true you can do:
- Возможность изменять ядро (загружать модули, настраивать переменные ядра).
- Доступ ко всем устройствам на уровне ОС (чтение-запись устройств, включая диски, сетевые карты и прочее).
- Получение полного доступа к большинству операций уровня ядра, например, изменение приоритета процессов, выполнение управляющих сигналов (например, kill), перезагрузка компьютера и многое другое.
- Завершение процессов хоста.
- Изменение поведения ядра (загрузка модулей, изменение ядерных параметров).
- Работа с низкоуровневыми аппаратными ресурсами.
- Полноценный контроль процессов, сетей и устройств хоста.

without privilige true
- Без режима privileged, вы не сможете воздействовать на процессы хоста напрямую (например, завершить процесс, изменить приоритет, остановить его и т.п.).
- Ошибки, подобные "Operation not permitted", возникнут, если вы попытаетесь вмешиваться в процессы хоста без повышения привилегий.
